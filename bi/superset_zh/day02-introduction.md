## 用户故事1：导航Superset界面

**标题**：作为一名新用户，我希望理解Superset界面，以便能够有效地导航应用程序。

**描述**：
Superset拥有一个全面的Web界面，包含多个不同功能的部分。理解布局有助于用户高效工作。

**验收标准**：
- [ ] 用户可以访问所有主要部分
- [ ] 导航菜单功能正常
- [ ] 用户理解每个部分的用途
- [ ] 界面响应迅速且用户友好

**分步指南**：

1. **访问Superset**
   - 打开浏览器：http://localhost:8088
   - 使用管理员凭据登录（admin/admin）

2. **探索主要部分**

3. **理解导航**
   - 顶部导航栏包含主要部分
   - 左侧边栏显示最近的项目
   - 面包屑显示当前位置
   - 提供搜索功能

4. **关键界面元素**
   - **+** 按钮：创建新项目
   - **设置** 齿轮：配置选项
   - **用户菜单**：个人资料和注销
   - **帮助**：文档和支持

## 用户故事2：理解数据源

**标题**：作为一名数据分析师，我希望理解Superset中的数据源如何工作，以便能够连接到我的数据库。

**描述**：
数据源是Superset的基础。理解如何连接和管理数据源对于创建可视化至关重要。

**验收标准**：
- [ ] 用户理解数据库连接
- [ ] 用户可以查看可用的数据集
- [ ] 用户理解数据源类型
- [ ] 用户可以探索数据结构

**分步指南**：

1. **访问数据源**
   - 导航到数据→数据库
   - 查看已连接数据库列表
   - 检查连接状态

2. **探索数据集**
   - 导航到数据→数据集

4. **测试连接**
   - 点击数据库的"测试连接"

## 用户故事3：创建第一个图表

**标题**：作为一名业务用户，我希望在Superset中创建第一个图表，以便能够可视化数据并获得洞察。

**描述**：
图表是仪表板的构建块。创建第一个图表有助于理解可视化过程和数据探索。

**验收标准**：
- [ ] 用户可以访问图表创建界面
- [ ] 用户可以选择数据源
- [ ] 用户可以选择图表类型
- [ ] 用户可以配置图表属性
- [ ] 图表正确显示

**分步指南**：

1. **开始创建图表**
   - 点击"+"按钮→"图表"
   - 或导航到图表→+图表

2. **选择数据源**
   - 从可用数据集中选择
   - 选择"世界银行健康统计数据"（示例数据集）
   - 点击"创建图表"

3. **选择图表类型**
   - 为第一个图表选择"条形图"
   - 其他选项：折线图、饼图、散点图等

4. **配置图表**
   - **指标**：选择要度量的内容（例如，SUM(population)）
   - **分组依据**：选择维度（例如，country）
   - **过滤器**：添加任何数据过滤器
   - **时间范围**：如果适用，设置时间段

5. **保存图表**
   - 点击"保存"按钮
   - 输入图表名称："按国家划分的人口"
   - 选择仪表板或创建新仪表板

**参考文档**：
- [图表创建指南](https://superset.apache.org/docs/creating-charts-dashboards/creating-charts)
- [图表类型参考](https://superset.apache.org/docs/creating-charts-dashboards/exploring-charts)

## 用户故事4：探索SQL实验室

**标题**：作为一名SQL开发人员，我希望探索SQL实验室，以便能够编写和测试用于数据分析的SQL查询。

**描述**：
SQL实验室提供了一个强大的界面，用于编写、测试和保存SQL查询。它对于复杂的数据分析和自定义数据集至关重要。

**验收标准**：
- [ ] 用户可以访问SQL实验室
- [ ] 用户可以编写SQL查询
- [ ] 用户可以执行查询
- [ ] 用户可以查看查询结果
- [ ] 用户可以保存查询

**分步指南**：

1. **访问SQL实验室**
   - 导航到SQL实验室→SQL编辑器
   - 或点击顶部导航中的"SQL实验室"

2. **选择数据库**
   - 从下拉菜单中选择数据库
   - 如果适用，选择模式/数据库
   - 验证连接状态

3. **编写简单查询**
   ```sql
   SELECT 
       country,
       SUM(population) as total_population
   FROM world_bank_population
   WHERE year = 2020
   GROUP BY country
   ORDER BY total_population DESC
   LIMIT 10
   ```

4. **执行查询**
   - 点击"运行"按钮
   - 等待结果加载
   - 审查查询性能

5. **保存查询**
   - 点击"保存"按钮
   - 输入查询名称："按人口排名前十的国家"
   - 如需要，添加描述

**参考文档**：
- [SQL实验室指南](https://superset.apache.org/docs/using-superset/sql-lab)
- [SQL最佳实践](https://superset.apache.org/docs/using-superset/sql-lab-best-practices)

## 用户故事5：理解用户角色和权限

**标题**：作为一名管理员，我希望理解Superset中的用户角色和权限，以便能够有效地管理访问控制。

**描述**：
Superset拥有一套全面的基于角色的访问控制系统，决定了用户在应用程序内可以看到和做什么。

**验收标准**：
- [ ] 用户理解不同的角色
- [ ] 用户可以查看角色权限
- [ ] 用户理解访问控制
- [ ] 用户可以管理基本权限

**分步指南**：

1. **访问安全设置**
   - 导航到安全→用户
   - 查看用户列表及其角色
   - 点击特定用户查看详细信息

2. **理解默认角色**
   - **管理员**：对所有功能的完全访问权限
   - **Alpha**：可以访问所有数据源
   - **Gamma**：对特定数据源的有限访问权限
   - **公共**：无访问权限（新用户的默认设置）

3. **查看角色权限**
   - 导航到安全→角色
   - 点击角色查看权限
   - 理解权限类别：
     - **数据源访问**：哪些数据库/表
     - **仪表板访问**：哪些仪表板
     - **图表访问**：哪些图表
     - **功能访问**：哪些功能

4. **创建自定义角色**
   - 点击"添加角色"
   - 输入角色名称："分析师"
   - 选择适当的权限
   - 保存角色

**参考文档**：
- [安全性和权限](https://superset.apache.org/docs/using-superset/security)
- [基于角色的访问控制](https://superset.apache.org/docs/using-superset/security#role-based-access-control)

## 关键概念摘要

### 1. 数据层次结构
- **数据库**：数据源连接
- **数据集**：来自数据库的表和视图
- **图表**：基于数据集的可视化
- **仪表板**：图表集合

### 2. 用户界面
- **顶部导航**：主要部分和功能
- **左侧边栏**：最近的项目和快速访问
- **内容区域**：主要工作区
- **设置**：配置和偏好

### 3. 工作流程
- 连接数据源
- 创建图表和可视化
- 构建仪表板
- 分享和协作

### 4. 最佳实践
- 为图表和仪表板使用描述性名称
- 在创建图表之前测试查询
- 逻辑地组织仪表板
- 记录数据源和假设
