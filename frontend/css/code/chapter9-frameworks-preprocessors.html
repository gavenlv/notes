模拟Sass混合生成的卡片组件>

卡片标题

这是一个使用Sass混合（Mixins）和变量生成的卡片组件。通过预处理器可以轻松实现样式的复用和定制。

// Sass变量定义
$primary-color: #3498db;
$secondary-color: #2ecc71;
$border-radius: 4px;
$transition-speed: 0.3s;

// Sass混合定义
@mixin button-base {
  display: inline-block;
  padding: 8px 16px;
  margin-right: 10px;
  margin-bottom: 10px;
  border: none;
  border-radius: $border-radius;
  font-size: 14px;
  font-weight: 500;
  text-align: center;
  text-decoration: none;
  cursor: pointer;
  transition: all $transition-speed ease;
  
  &:hover {
    transform: translateY(-2px);
  }
}

@mixin button-variant($bg-color, $hover-color) {
  background-color: $bg-color;
  color: white;
  
  &:hover {
    background-color: $hover-color;
    box-shadow: 0 4px 8px rgba($bg-color, 0.3);
  }
}

// 使用混合生成样式
.btn-custom {
  @include button-base;
}

.btn-custom-primary {
  @include button-variant($primary-color, darken($primary-color, 10%));
}

.btn-custom-secondary {
  @include button-variant($secondary-color, darken($secondary-color, 10%));
}

// Sass嵌套示例
.nested-nav {
  background-color: $primary-color;
  padding: 10px;
  border-radius: $border-radius;
  
  ul {
    list-style: none;
    padding: 0;
    margin: 0;
    display: flex;
    gap: 10px;
    
    li {
      display: inline-block;
      
      a {
        color: white;
        text-decoration: none;
        padding: 8px 12px;
        border-radius: $border-radius;
        transition: background-color $transition-speed ease;
        
        &:hover {
          background-color: rgba(255, 255, 255, 0.2);
        }
        
        &.active {
          background-color: rgba(255, 255, 255, 0.3);
          font-weight: 600;
        }
      }
    }
  }
}

4. CSS变量与主题切换示例

使用CSS变量实现的主题切换功能> <

主题演示区域

这个区域的背景、文字和边框颜色会随着主题的切换而改变，完全基于CSS变量实现。

嵌套内容也会自动适应主题变化 
// CSS变量定义
:root {
    --primary-color: #3498db;
    --secondary-color: #2ecc71;
    --border-radius: 4px;
    --transition-speed: 0.3s;
}

// 主题定义
.theme-light {
    --theme-bg: #ffffff;
    --theme-text: #333333;
    --theme-border: #dddddd;
}

.theme-dark {
    --theme-bg: #1a1a1a;
    --theme-text: #f0f0f0;
    --theme-border: #444444;
}

// 使用CSS变量
.theme-demo {
    padding: 20px;
    background-color: var(--theme-bg);
    color: var(--theme-text);
    border: 1px solid var(--theme-border);
    border-radius: var(--border-radius);
    transition: all var(--transition-speed) ease;
}

// JavaScript主题切换
const themeToggle = document.getElementById('themeToggle');
const body = document.body;
let isDarkTheme = false;

themeToggle.addEventListener('click', () => {
    isDarkTheme = !isDarkTheme;
    
    if (isDarkTheme) {
        body.classList.remove('theme-light');
        body.classList.add('theme-dark');
        themeToggle.innerHTML = '\u003ci class="fa fa-sun-o" aria-hidden="true"\u003e\u003c/i\u003e 切换到浅色主题';
    } else {
        body.classList.remove('theme-dark');
        body.classList.add('theme-light');
        themeToggle.innerHTML = '\u003ci class="fa fa-moon-o" aria-hidden="true"\u003e\u003c/i\u003e 切换到深色主题';
    }
});

6. 构建工具集成示例

Webpack和PostCSS配置示例> < 
// webpack.config.js
module.exports = {
  module: {
    rules: [
      {
        test: /\.(scss|sass)$/,
        use: [
          'style-loader', // 将CSS注入到DOM
          'css-loader',   // 解析CSS文件
          {
            loader: 'postcss-loader',
            options: {
              postcssOptions: {
                plugins: ['autoprefixer']
              }
            }
          },
          'sass-loader'   // 编译SCSS到CSS
        ]
      }
    ]
  }
}

// postcss.config.js
module.exports = {
  plugins: [
    require('autoprefixer'),
    require('postcss-preset-env')({
      stage: 3,
      features: {
        'custom-properties': false
      }
    }),
    require('cssnano')({
      preset: 'default'
    })
  ]
}

Vite配置示例> 
// vite.config.js
import { defineConfig } from 'vite'

export default defineConfig({

      ]
    },

      scss: {
        additionalData: `@import "${__dirname}/src/styles/variables.scss";`
      }
    }
  }

7. CSS框架与预处理器最佳实践总结

选择合适的CSS技术栈

技术/框架 适用场景 优势 注意事项
Bootstrap 企业网站、管理后台、快速原型 组件丰富、文档完善、社区支持强 文件体积较大、定制化需要覆盖样式
Tailwind CSS 需要高度定制化的界面、快速开发 开发效率高、无命名冲突、体积可控 HTML类名较多、需要熟悉工具类
Sass/SCSS 大型项目、需要复用样式 功能丰富、结构清晰、易于维护 需要编译步骤、学习曲线较陡
PostCSS 需要特定后处理功能的项目 灵活、可插拔、模块化 配置复杂、需要选择合适的插件
CSS-in-JS React/Vue等现代前端框架项目 组件级样式封装、动态样式能力强 运行时开销、调试困难

性能优化建议
  • 使用PurgeCSS移除未使用的CSS
  • 按需引入框架组件，减少体积
  • 利用CSS Modules或命名空间避免冲突
  • 优化CSS选择器，避免复杂嵌套
  • 压缩CSS文件，减少传输大小
  • 使用content-visibility提升渲染性能

可维护性最佳实践

  • 建立清晰的文件结构和命名约定
  • 使用变量管理颜色、字体、间距等设计系统
  • 封装可复用的样式为混合或组件
  • 编写有意义的注释，解释复杂样式
  • 定期重构和优化CSS代码
  • 在团队中统一CSS编码规范
