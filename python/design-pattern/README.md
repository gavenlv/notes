## 设计模式分类

### 创建型模式

创建型模式关注对象的创建过程，将对象的创建和使用分离。

1. [单例模式](2-单例模式.md): 确保一个类只有一个实例，并提供全局访问点。
2. [工厂方法模式](3-工厂方法模式.md): 定义了一个创建对象的接口，但让子类决定实例化哪一个类。
3. [建造者模式](4-建造者模式.md): 将一个复杂对象的构建与其表示分离。
4. [原型模式](5-原型模式.md): 通过复制现有对象来创建新对象。

### 结构型模式

结构型模式关注类和对象的组合，如何将它们组合成更大的结构。

5. [适配器模式](6-适配器模式.md): 将一个类的接口转换成客户希望的另一个接口。
6. [桥接模式](7-桥接模式.md): 将抽象部分与实现部分分离，使它们都可以独立地变化。
7. [组合模式](8-组合模式.md): 组合多个对象形成树形结构以表示"部分-整体"的层次结构。
8. [装饰器模式](9-装饰器模式.md): 动态地给一个对象添加一些额外的职责。
9. [外观模式](10-外观模式.md): 为子系统中的一组接口提供一个统一的界面。
10. [享元模式](11-享元模式.md): 运用共享技术有效地支持大量细粒度的对象。
11. [代理模式](12-代理模式.md): 为其他对象提供一种代理以控制对这个对象的访问。

### 行为型模式

行为型模式关注对象之间的职责分配和通信。

12. [责任链模式](13-责任链模式.md): 使多个对象都有机会处理请求，从而避免请求的发送者和接收者之间的耦合关系。
13. [命令模式](14-命令模式.md): 将一个请求封装为一个对象，从而使你可用不同的请求对客户进行参数化。
14. [解释器模式](15-解释器模式.md): 给定一种语言，定义它的文法的一种表示，并定义一个解释器。
15. [迭代器模式](16-迭代器模式.md): 提供一种方法顺序访问一个聚合对象中各个元素，而又不需暴露该对象的内部表示。
16. [中介者模式](17-中介者模式.md): 用一个中介对象来封装一系列的对象交互。
17. [备忘录模式](18-备忘录模式.md): 在不破坏封装性的前提下，捕获一个对象的内部状态。
18. [观察者模式](19-观察者模式.md): 定义对象间的一种一对多的依赖关系。
19. [状态模式](20-状态模式.md): 允许一个对象在其内部状态改变时改变它的行为。
20. [策略模式](21-策略模式.md): 定义一系列的算法,把它们一个个封装起来。
21. [模板方法模式](22-模板方法模式.md): 定义一个操作中的算法的骨架，而将一些步骤延迟到子类中。
22. [访问者模式](23-访问者模式.md): 表示一个作用于某对象结构中的各元素的操作。

## 学习路径

1. **基础阶段**: 从[设计模式基础概念](1-设计模式基础概念.md)开始，了解设计模式的背景和六大原则。
2. **创建型模式**: 学习[单例模式](2-单例模式.md)、[工厂方法模式](3-工厂方法模式.md)、[建造者模式](4-建造者模式.md)和[原型模式](5-原型模式.md)。
3. **结构型模式**: 学习[适配器模式](6-适配器模式.md)到[代理模式](12-代理模式.md)的七个结构型模式。
4. **行为型模式**: 学习[责任链模式](13-责任链模式.md)到[访问者模式](23-访问者模式.md)的十一个行为型模式。
5. **实战应用**: 学习[设计模式实战项目](24-设计模式实战项目.md)，了解如何在实际项目中综合应用多种设计模式。

## 最佳实践

1. **不要过度设计**: 不是所有地方都需要设计模式，只在确实需要时使用
2. **理解模式的本质**: 理解每个模式解决的问题，而不是记忆具体实现
3. **考虑组合使用**: 多个模式可以组合使用，解决更复杂的问题
4. **保持简单**: 在满足需求的前提下，保持实现的简单性
5. **文档化设计**: 记录设计决策和模式使用的原因
