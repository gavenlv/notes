## 常见问题解答

### 基础问题
- **Q: 单元测试和集成测试有什么区别？**
  A: 单元测试测试单个组件，不依赖外部系统；集成测试测试多个组件间的交互。

- **Q: JUnit 4和JUnit 5有什么区别？**
  A: JUnit 5采用模块化架构，提供更丰富的注解和扩展机制，支持参数化测试和动态测试。

- **Q: 什么时候应该使用Mock对象？**
  A: 当测试对象依赖外部系统（如数据库、网络服务）或依赖执行缓慢时。

### 进阶问题
- **Q: 如何提高测试覆盖率？**
  A: 首先关注关键业务逻辑，然后逐步覆盖边界条件和异常情况，避免为了覆盖率而编写无意义的测试。

- **Q: TDD适合所有项目吗？**
  A: TDD特别适合需求明确、逻辑复杂的业务系统，对于探索性或原型开发可能不太适用。

- **Q: 如何测试私有方法？**
  A: 通常不建议直接测试私有方法，而是通过测试公共方法间接验证私有方法的正确性。

### 专家问题
- **Q: 如何在微服务架构中组织测试？**
  A: 采用测试金字塔策略：单元测试、服务测试、契约测试、端到端测试。

- **Q: 如何管理测试数据？**
  A: 使用Builder模式或工厂模式创建测试数据，考虑使用测试容器管理集成测试数据。
