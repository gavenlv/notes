# 第4章 — 参数化测试与表驱动测试

目标：
- 掌握 pytest 的参数化（`@pytest.mark.parametrize`）。
- 使用表驱动方法组织多个输入/期望对，减少重复测试代码。

概念解析
- 参数化：一次性定义多组输入/期望，pytest 会为每组数据生成一个测试用例。
- 表驱动测试：把输入/输出组合放在数据表（list/tuple/dict）中，遍历执行相同断言。

示例代码位于 `testing/unit-test/code/chapter4/`。

示例（判断回文字符串）：

- `is_palindrome.py` — 包含业务函数。
- `test_is_palindrome.py` — 使用 `@pytest.mark.parametrize` 测试多个情形。

最佳实践
- 把边界条件和常见用例都放到参数化表里。
- 当有很多组合时把表单独抽出来（便于维护）。

练习
- 为空串、单字符、包含空格和标点的回文添加测试用例。
