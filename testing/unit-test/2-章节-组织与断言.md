# 第2章 — 测试组织与断言

目标：
- 学会如何组织测试文件和命名测试。
- 掌握常见断言（断言风格）以及何时使用哪种断言。

概念解析
- 文件结构约定：对于模块 `mypkg/foo.py`，测试通常放 `tests/test_foo.py` 或 `tests/foo/test_*.py`。
- 名称约定：测试模块/文件以 `test_` 开头，测试函数以 `test_` 开头，测试类以 `Test` 开头并且不要有自定义构造函数。

常见断言（pytest 风格）
- assert a == b
- assert a in b
- with pytest.raises(SomeError): ...

示例：
- 组织示例在 `testing/unit-test/code/chapter2/`。

代码说明见章节代码（已经包含示例）。

最佳实践
- 把测试按功能而不是实现组织。
- 避免在测试中写复杂的逻辑；测试应简单明了。
- 使用有描述性的断言信息或通过断言拆分来获得更好的失败输出。
