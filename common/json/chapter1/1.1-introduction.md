# 1.1 JSON简介与历史

## 什么是JSON

JSON（JavaScript Object Notation，JavaScript对象表示法）是一种轻量级的数据交换格式。它基于JavaScript的一个子集，但JSON是完全独立于语言的文本格式，使用类似C语言家族的习惯（包括C、C++、C#、Java、JavaScript、Perl、Python等）。

### JSON的核心特点

1. **轻量级**：相比XML，JSON更加简洁，占用更少的空间
2. **易于人阅读和编写**：JSON的语法直观，便于开发人员阅读和编辑
3. **易于机器解析和生成**：所有主流编程语言都支持JSON解析和生成
4. **语言无关性**：虽然源自JavaScript，但现在几乎所有编程语言都支持JSON
5. **自描述性**：数据结构清晰，易于理解

## JSON的历史

### 起源

- **1999年**：JSON的概念起源于JavaScript，最初由Douglas Crockford提出
- **2002年**：Crockford在state.com网站上正式定义了JSON格式
- **2005年**：JSON开始获得广泛关注，被Yahoo!等大型互联网公司采用
- **2006年**：RFC 4627发布，JSON被正式定义为互联网标准
- **2013年**：RFC 7159更新了JSON标准
- **2017年**：RFC 8259进一步更新并替代了RFC 7159

### JSON标准的演进

| 标准版本 | 发布年份 | 主要特点 |
|---------|---------|----------|
| RFC 4627 | 2006 | 首次正式定义JSON标准 |
| RFC 7159 | 2013 | 修正错误，明确JSON为文本格式 |
| RFC 8259 | 2017 | 进一步明确规范，解决歧义 |

## 为什么选择JSON

### 与XML的比较

在过去，XML是数据交换的主流格式。现在，JSON在许多场景下已成为首选。以下是两者的简单比较：

| 特性 | JSON | XML |
|------|------|-----|
| 可读性 | 高 | 较高 |
| 解析难度 | 简单 | 复杂 |
| 数据体积 | 小 | 大 |
| 与JavaScript的兼容性 | 原生支持 | 需要额外解析 |
| 对象表示 | 直接 | 间接 |
| 注释 | 不支持 | 支持 |

### JSON的优势实例

**JSON示例**：
```json
{
  "name": "张三",
  "age": 30,
  "isStudent": false,
  "courses": ["数学", "物理", "化学"],
  "address": {
    "street": "人民路123号",
    "city": "北京",
    "zipCode": "100000"
  }
}
```

**等效的XML示例**：
```xml
<person>
  <name>张三</name>
  <age>30</age>
  <isStudent>false</isStudent>
  <courses>
    <course>数学</course>
    <course>物理</course>
    <course>化学</course>
  </courses>
  <address>
    <street>人民路123号</street>
    <city>北京</city>
    <zipCode>100000</zipCode>
  </address>
</person>
```

从以上示例可以看出，JSON更加简洁，且更接近于编程语言中的对象表示方式。

## JSON的基本结构

JSON构建在两种结构之上：

1. **名称/值对的集合**（Object）：在各种语言中，这被实现为对象、记录、结构、字典、哈希表、有键列表或关联数组
2. **值的有序列表**（Array）：在大多数语言中，这被实现为数组、向量、列表或序列

这些是常见的数据结构。事实上，所有现代编程语言都以某种形式支持这些数据结构。这使得一种数据格式在同样基于这些结构的编程语言之间交换成为可能。

## JSON的应用场景

JSON广泛应用于以下领域：

1. **Web API**：作为RESTful API的标准数据格式
2. **配置文件**：应用程序和系统配置
3. **数据存储**：NoSQL数据库如MongoDB使用JSON格式存储数据
4. **前端与后端通信**：AJAX请求和响应
5. **微服务间通信**：服务间的数据交换
6. **物联网**：设备与服务器间的数据传输

## JSON的局限性

尽管JSON有很多优点，但它也有一些局限性：

1. **不支持注释**：JSON不支持注释，使得配置文件的可读性降低
2. **数据类型有限**：不支持日期、函数等复杂数据类型
3. **没有验证机制**：本身不提供数据验证功能（需要借助JSON Schema）
4. **严格的语法**：尾随逗号等常见写法不被支持

## 实验环节：查看不同的JSON示例

让我们通过一些简单的示例来直观地了解JSON的结构：

### 示例1：简单的键值对
```json
{
  "username": "admin",
  "password": "123456",
  "isActive": true
}
```

### 示例2：包含数组的JSON
```json
{
  "fruits": ["apple", "banana", "orange"],
  "numbers": [1, 2, 3, 4, 5]
}
```

### 示例3：嵌套的复杂对象
```json
{
  "company": "TechCorp",
  "employees": [
    {
      "id": 1,
      "name": "张三",
      "position": "工程师",
      "skills": ["Java", "Python", "JavaScript"]
    },
    {
      "id": 2,
      "name": "李四",
      "position": "设计师",
      "skills": ["Photoshop", "Illustrator", "UI/UX"]
    }
  ]
}
```

这些示例展示了JSON的基本结构：使用花括号表示对象，使用方括号表示数组，键值对用冒号分隔，多个键值对之间用逗号分隔。
